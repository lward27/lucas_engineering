# Agent Topology

Date: 2026-05-10

Phase 1 uses the desktop-hosted Hermes instance as the only active worker runtime. Codex remains the top-level orchestrator.

## Control Flow

```text
User goal
  -> Codex orchestrator
  -> Hermes planning/review/observability/release agents
  -> Skills
  -> GitHub, Argo CD, Kubernetes, Grafana, Prometheus, Mimir, Tempo
  -> Codex final decision
```

## Registered Agents

| Agent | Status | Purpose | Tool grant |
| --- | --- | --- | --- |
| `hermes-planning` | active | Goal intake, requirements, specs, task decomposition | `hermes-readonly-planning` |
| `hermes-reviewer` | active | Code, manifest, architecture, and risk review | `hermes-readonly-review` |
| `hermes-observability` | active | Argo/Kubernetes/metrics/traces/dashboard observation | `hermes-readonly-observability` |
| `hermes-gitops-release` | planned | GitOps release PR proposal | `hermes-gitops-proposer` |

## Discovery

Codex discovers agents from:

```text
agent-skills/agents/registry/agents.yaml
```

The registry declares:

- runtime
- endpoint
- service account
- allowed skills
- tool grant
- context policy
- secret policy
- concurrency
- audit sink

## Runtime Boundary

Hermes runs on the desktop host at `192.168.50.145`, with Kubernetes providing the public ingress/front door through the `hermes-agent` Service and EndpointSlice.

The registry points agent dispatch at the desktop Hermes API URL. Kubernetes RBAC does not automatically flow into the desktop process; the desktop process must use its local kubeconfig or the short-lived kubeconfig generated by `agent-skills/scripts/generate-desktop-hermes-kubeconfig.rb`.

## What Agents Must Not Do

- bypass `permission-checker`
- bypass `approval-gate-manager`
- read raw secret values
- mutate live Kubernetes resources directly
- sync Argo CD without approval
- publish images without approval
- operate outside registered capabilities

## Future Extension

Custom harnesses may be added later by extending the registry and creating scoped RBAC, network policy, health checks, and audit logging. They should not be active in phase 1.
